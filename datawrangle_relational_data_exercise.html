This is a mutating join, and the basic function you need to use here is left_join(). We have to perform the joining operation twice since we want to create new variables based on both the destination airport and the origin airport. And because the name of the key variable differs between the data frames, we need to explicitly define how to join the data frames using the by argument:

This is better, but now we have two sets of latitude and longitude variables in the data frame: one for the destination airport, and one for the origin airport. When we perform the second left_join() operation, to avoid duplicate variable names the function automatically adds generic .x and .y suffixes to the output to disambiguate them. This is nice, but we might want something more intuitive to explicitly identify which variables are associated with the destination vs.Â the origin. To do that, we override the default suffix argument with custom suffixes:

airports_lite <- airports %>%
  select(faa, lat, lon)

flights %>%
  left_join(airports_lite, by = c(dest = "faa")) %>%
  left_join(airports_lite, by = c(origin = "faa"), suffix = c(".dest", ".origin"))

Is there a relationship between the age of a plane and its departure delays?

Hint: all the data is from 2013.

Click for the solution

The first step is to calculate the age of each plane. To do that, use planes and the age variable:

plane_ages <- planes %>%
  mutate(age = 2013 - year) %>%
  select(tailnum, age)

The best approach to answering this question is a visualization. There are several different types of visualizations you could implement (e.g.Â scatterplot with smoothing line, line graph of average delay by age). The important thing is that we need to combine flights with plane_ages to determine for each flight the age of the plane. This is another mutating join. The best choice is inner_join() as this will automatically remove any rows in flights where we donât have age data on the plane.

# smoothing line
flights %>%
  inner_join(plane_ages) %>%
  ggplot(aes(age, dep_delay)) +
  geom_smooth()
## Joining, by = "tailnum"
## `geom_smooth()` using method = 'gam'
## Warning: Removed 9374 rows containing non-finite values (stat_smooth).

# line graph of average delay by age
flights %>%
  inner_join(plane_ages) %>%
  group_by(age) %>%
  summarise(delay = mean(dep_delay, na.rm = TRUE)) %>%
  ggplot(aes(age, delay)) +
  geom_point() +
  geom_line()
## Joining, by = "tailnum"
## Warning: Removed 1 rows containing missing values (geom_point).

The important takeaway is that departure delays do not appear to increase with plane age â in fact they seem to decrease slightly (though with an expanding confidence interval). Care to think of a reason why this may be so?

What does it mean for a flight to have a missing tailnum? What do the tail numbers that donât have a matching record in planes have in common?

Hint: one variable explains ~90% of the problems.

Click for the solution

To answer this question, we need to figure out what flights with missing tail numbers have in common. To do this, we need to filter flights and remove observations with missing tail numbers. But tailnum is stored in planes, so we need to join the data frames together first. Because we donât care about the actual value of tailnum (just that it is missing), we can use a filtering join to merge the data frames. Specifically, anti_join() will filter flights and keep only observations where we do not have the planeâs tail number.

flights %>%

Hmm, did this actually remove all the observations we intended? Actually not. By default anti_join() will merge the data frames based on all common column names. Here, both data frames have columns for year and tailnum. But they represent different variables. In flights, year indicates the year in which the flight departed. But in planes, year represents the manufacturing year. So to make anti_join() do what we intended, we need to manually specify that it should only match observations based on tailnum.

  anti_join(planes, by = "tailnum")

Filter flights to only show flights with planes that have flown at least 100 flights.

Click for the solution

First we need to determine which planes have flown at least 100 flights. This is a simple count() function using flights, summarizing the number of flights by tailnum and filtering the resulting data frame to only keep tail numbers for planes with more than 100 flights. Then because we want to filter flights, we need a filtering join operation to merge the summarized data frame with the original flights object. This calls for a semi_join() operation.

planes_gt100 <- flights %>%
  count(tailnum) %>%
  filter(n > 100)

flights %>%
  semi_join(planes_gt100)
